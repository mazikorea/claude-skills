# 서베이 설계 스킬

## 핵심 원칙

서베이의 목적은 **고객의 진짜 니즈를 왜곡 없이 파악**하는 것.
"우리가 듣고 싶은 답"이 아니라 "고객이 실제로 생각하는 것"을 얻어야 한다.

### 3가지 불변 원칙
1. **질문이 답을 유도하면 안 됨** — 선택지, 문구, 순서 모두 중립적이어야
2. **비교 가능해야 함** — 같은 목적의 서베이는 핵심 문항을 동일하게 유지 → 시계열 비교
3. **결과로 행동할 수 있어야 함** — "흥미롭다"로 끝나면 실패. "그래서 뭘 하지?"가 나와야 성공

---

## Step 1. 서베이 시작 전 — 목적 정의

서베이 설계 전에 반드시 아래 3가지를 확정한다.

### 1-1. 의사결정 유형 파악

"이 서베이 결과로 어떤 결정을 내릴 것인가?"

| 의사결정 | 설명 | 핵심 질문 |
|---------|------|----------|
| **시장 이해** | 새 카테고리/시장 진입 전 구조 파악 | 고객이 뭘 원하고, 뭐가 부족하고, 누가 경쟁자인가? |
| **컨셉 검증** | 기획한 제품/서비스의 시장 반응 확인 | 이 컨셉이 고객 니즈에 맞는가? 얼마를 낼 의향이 있는가? |
| **브랜드 건강** | 브랜드 인지도·이미지·포지션 변화 추적 | 시장에서 우리를 어떻게 보는가? 변하고 있는가? |
| **고객 만족** | 구매 고객의 만족·불만·재구매 의향 파악 | 만족하는가? 뭘 고쳐야 하는가? |

→ 한 서베이가 여러 의사결정을 커버할 수 있음. 단, **주 목적 1개**를 반드시 정해야 문항 우선순위가 잡힘.

### 1-2. 대상 확정

| 대상 | 언제 쓰는가 | 모집 방법 |
|------|-----------|----------|
| 시장 전체 (비고객 포함) | 시장 이해, 컨셉 검증, 브랜드 건강 | 오픈서베이 등 패널 구매 |
| 자사 고객 | 고객 만족, 재구매 의향 | 자사 DB (자사몰, CRM) |

### 1-3. 결과물 사전 정의

서베이 결과로 **어떤 산출물**을 만들 것인지 미리 정한다.
산출물을 모르면 질문을 설계할 수 없음.

| 산출물 예시 | 필요한 질문 모듈 |
|-----------|----------------|
| 경쟁 브랜드 포지셔닝 맵 | 브랜드 퍼널 (인지→고려→사용) |
| 구매 결정 요인 순위표 | 결정 요인 (순위형) |
| 미충족 니즈 목록 | 현재 불만점 (주관식) |
| 적정 가격 범위 (PSM 곡선) | 가격 민감도 4문항 |
| 채널 우선순위 | 구매 채널 + 정보 탐색 채널 |
| NPS 추이 | 추천 의향 (0~10 척도) |
| 컨셉 구매 의향 | 컨셉 제시 → 구매 의향 척도 |

## Step 2. 질문 모듈 — 목적별 조합

### 모듈 목록

각 모듈은 독립적으로 조합 가능. 의사결정 유형에 따라 필요한 모듈을 선택한다.

#### M1. 스크리닝 (자격 필터)
- **용도**: 타겟이 아닌 응답자 걸러내기
- **필수 항목**: 성별, 연령대, 직업군 (교차분석용)
- **필터 질문**: 해당 카테고리 구매 경험 or 구매 의향 (없으면 종료)
- **문항 수**: 3~5문항 이내. 길면 응답자 이탈
- **정답 유추 방지**: 질문에서 "어떤 답이 통과인지" 추론 불가능하게 설계. 타겟 항목을 허위 항목 사이에 배치 (예: 브랜드 10개 중 타겟 3개 섞기)
- **주의**: 스크리닝 조건이 너무 좁으면 응답자 모집 어려움 → 패널 업체와 사전 협의

#### M2. 브랜드 퍼널 (인지 → 숙지 → 고려 → 첫구매 → 사용 → 충성)
- **용도**: 경쟁 구도 파악, 자사 브랜드 포지션 측정, 퍼널 전환율 분석
- **구성** (6단계):
  1. 비보조 인지도 (주관식) — **반드시 서베이 첫 질문**. 보기 노출 전에 배치
     - "이 카테고리 하면 떠오르는 브랜드를 자유롭게 적어주세요"
  2. 보조 인지도 (복수선택, 랜덤 순서) — 브랜드 리스트 제시
     - "다음 브랜드 중 알고 있는 것을 모두 선택해주세요"
  3. 숙지도 (복수선택) — 이름만 아는 것과 내용을 아는 것 구분
     - "다음 브랜드 중 어떤 제품을 파는지 알고 있는 브랜드를 선택해주세요"
  4. 고려도 (복수선택) — "실제 구매를 고려할 브랜드"
  5. 사용 경험 (복수선택) — "구매한 적 있는 브랜드" (첫구매 여부 확인)
  6. 주 사용 (단일선택 or 주관식) — "가장 자주 사용하는 브랜드" (충성도)
- **시계열 규칙**: 보조 인지도의 브랜드 리스트는 한번 정하면 유지. 추가는 가능하되 삭제 금지.
- **간소화 옵션**: 서베이 시간이 부족하면 3·5·6단계 중 선택적 생략 가능. 단, 1·2·4단계는 필수.

#### M3. 구매 결정 요인
- **용도**: 고객이 제품을 고를 때 뭘 기준으로 하는지 우선순위 파악
- **형식 선택 기준**:
  - 보기 5개 이하 → **순위형** (상위 3개를 순서대로 선택)
  - 보기 6개 이상 → **MaxDiff** 권장 (각 세트에서 "가장 중요"/"가장 안 중요" 1개씩 선택)
  - 플랫폼이 MaxDiff 미지원 시 → "상위 3개를 순서대로 선택" 순위형으로 대체. 단, 7개 초과 항목의 중간 순위 데이터는 신뢰도 낮음을 인지
- **절대 쓰면 안 되는 형식**: "최대 N개 복수선택" → 변별력 없음. 모든 항목이 비슷하게 높게 나옴
- **보기 설계 원칙**:
  - 카테고리에 영향을 주는 모든 요인을 빠짐없이 포함 (MECE)
  - "가격"을 빼면 안 됨 — 기존 서베이에서 가장 흔한 실수
  - 보기는 반드시 랜덤 순서로 제시
  - 보기 항목은 8~12개가 적정 (너무 적으면 누락, 너무 많으면 인지 부하)
- **시계열 규칙**: 보기 항목을 동일하게 유지 → 시점별 변화 추적 가능

#### M4. 미충족 니즈 (불만·개선점)
- **용도**: 시장의 빈틈 = 차별화 기회 포착
- **형식**: **주관식** (자유 응답)
- **왜 주관식인가**: 선택지를 주면 선택지에 있는 것만 높게 나옴 → 진짜 불만을 놓침
- **질문 예시**:
  - "현재 사용 중인 [카테고리] 제품에서 가장 불만족스러운 점은?"
  - "[카테고리] 제품을 새로 산다면 반드시 있어야 하는 기능 1가지는?"
- **분석 방법**: 주관식 응답을 카테고리화 → 빈도 집계 → 상위 5개 = 핵심 미충족 니즈
- **후속 활용**: 1차 서베이 주관식 결과의 상위 항목을 → 2차 서베이에서 객관식 보기로 전환 → 정량 검증

#### M5. 가격 민감도 (PSM — Van Westendorp)
- **용도**: 시장이 수용하는 가격 범위와 적정 가격 도출
- **형식**: 숫자 직접 입력 (만원 단위)
- **필수 4문항** (순서 고정, 문구 변경 금지):
  1. "이 가격이면 **싸다**고 느끼는 금액은?"
  2. "**적정하다**고 느끼는 금액은?"
  3. "**비싸다**고 느끼는 금액은?"
  4. "**너무 비싸서 절대 안 살** 금액은?"
- **분석**: 4개 누적 분포 곡선의 교차점
  - OPP (최적 가격): "싸다" 곡선과 "비싸다" 곡선의 교차점
  - 수용 가격 범위: "너무 싸다"와 "너무 비싸다" 사이의 영역
- **주의**: 카테고리와 제품 형태를 명시해야 함 ("출퇴근용 가방" vs "가방" → 결과가 다름)
- **절대 쓰면 안 되는 형식**: "적정 가격대를 고르세요 (5~10만/10~15만/...)" → 구간 선택은 정밀도가 낮고, PSM 곡선을 그릴 수 없음

#### M6. 채널
- **용도**: 유통 전략과 마케팅 채널 우선순위 결정
- **반드시 2개로 분리**: 구매 채널 ≠ 정보 탐색 채널
  1. "주로 어디서 **구매**하시나요?" (복수선택, 최대 2개)
  2. "구매 전 정보를 어디서 **얻으시나요?**" (복수선택, 최대 2개)
- **보기**: 카테고리별 주요 채널 포함 (온라인 플랫폼, 자사몰, 오프라인, 검색 등)

#### M7. 컨셉 반응
- **용도**: 기획한 제품/서비스 컨셉에 대한 시장 반응 확인
- **구성**:
  1. 컨셉 제시 (이미지 + 설명문 — 중립적 톤, 과장 금지)
  2. 구매 의향 (5점 척도: 전혀 없다 ~ 매우 있다)
  3. 적정 가격 (숫자 입력)
  4. 개선 의견 (주관식: "이 제품에서 바꿨으면 하는 점은?")
- **주의**: 컨셉 설명에 브랜드 슬로건이나 긍정적 수식어 넣지 않기 → 유도 편향

#### M8. 만족도 / NPS
- **용도**: 구매 고객의 만족 수준과 추천 의향 측정
- **구성**:
  1. 전반적 만족도 (5점 척도)
  2. 만족하는 점 (주관식 or 순위형)
  3. 아쉬운 점 (주관식)
  4. NPS (0~10 척도: "타인에게 추천할 의향은?")
  5. **NPS 후속 질문 (주관식, 필수)**: "왜 그 점수를 주셨나요?" — 점수만으로는 "왜?"를 알 수 없음
  6. 재구매 의향 (5점 척도)
- **NPS 계산**: 추천(9~10) 비율 - 비추(0~6) 비율 = NPS
- **한국 시장 참고**: 한국 소비자는 극단 점수(9~10) 선택률이 낮아 NPS가 구조적으로 낮게 나올 수 있음. 글로벌 벤치마크와 직접 비교 시 주의. 국내 동종 업계 기준으로 비교하는 것이 적절.
- **시계열 규칙**: NPS 문항은 문구·척도를 절대 변경하지 않음

#### M9. 사용 실태
- **용도**: 고객이 제품을 실제로 어떻게 쓰는지 파악 (제품 기획용)
- **질문 예시**:
  - "이 카테고리 제품을 평균 몇 년에 한 번 구매하시나요?"
  - "[제품] 사용 시 꼭 넣고 다니는 것은?" (복수선택)
  - "어떤 형태의 제품을 사용하시나요?" (복수선택 or 단일선택)
- **주의**: 형태(백팩/토트 등)를 미리 한정하지 않기 → 고객이 실제로 쓰는 형태를 파악해야 함

#### M10. 자사 브랜드 인지 (시장 대상 서베이에서만 사용)
- **용도**: 비고객 포함 시장에서 자사 브랜드 인지도 및 이미지 측정
- **구성**:
  1. 인지 여부 (단일: 잘 알고 있다 / 들어본 적 있다 / 모른다)
  2. [인지 시] 이미지 (주관식 단문: "브랜드 하면 떠오르는 것")
  3. [인지 시] 구매 의향 (5점 척도)
  4. [비인지 시] 간단한 브랜드 소개 후 관심도 (5점 척도)
- **주의**: 브랜드 소개는 중립적으로. 슬로건·감성 문구 배제. "XX만원대 한국 [카테고리] 브랜드"식 팩트만.

### 질문 배치 순서 원칙

서베이 내 질문 배치는 아래 순서를 따른다 (Funnel approach: 일반적 → 구체적).

```
1. 스크리닝 (M1)
2. 비보조 인지도 (M2 첫 질문) — 반드시 맨 앞
3. 보조 인지도 + 숙지도 + 고려도 + 사용 (M2 나머지)
4. 사용 실태 — 행동 질문 (M9)
5. 구매 결정 요인 — 태도 질문 (M3)
6. 미충족 니즈 (M4)
7. 가격 (M5)
8. 채널 (M6)
9. 컨셉 반응 (M7) — 구체적 자극물 제시는 후반부
10. 자사 브랜드 인지 (M10)
11. 만족도/NPS (M8)
12. 인구통계 (성별/연령/직업군) — 항상 마지막
```

- **원칙**: 일반적 인식 → 구체적 평가 순서. 구체적 질문이 일반적 인식을 오염시키는 것 방지.
- **인구통계는 마지막**: 업계 공통 표준. 설문 초반에 놓으면 "조사당하는 느낌"으로 이탈 증가.
- **주의력 확인 질문**: 서베이 중간(40~60% 지점)에 1개 삽입 권장. "이 질문에서 '동의'를 선택해주세요" 등.

## Step 3. 질문 설계 체크리스트

### 설계 전

- [ ] 이 서베이의 **의사결정 유형**이 정해졌는가? (시장이해/컨셉검증/브랜드건강/고객만족)
- [ ] **주 목적 1개**가 명확한가?
- [ ] **대상**이 확정됐는가? (시장 전체 / 자사 고객)
- [ ] **결과 산출물**이 사전 정의됐는가? (어떤 표·그래프·리스트를 만들 것인지)
- [ ] 필요한 **모듈 조합**이 선택됐는가?
- [ ] 기존 서베이가 있다면, **고정 모듈 문항이 동일한지** 확인했는가?

### 설계 중

- [ ] 비보조 인지도(M2)는 **서베이 맨 앞**에 배치했는가?
- [ ] 복수선택 대신 **순위형**을 쓸 수 있는 질문은 전환했는가?
- [ ] 선택지에 **핵심 변수 누락**이 없는가? (특히 "가격" 빠뜨리지 않았는가?)
- [ ] 모든 객관식의 선택지 순서를 **랜덤화**했는가?
- [ ] PSM 4문항의 **문구와 순서**가 표준을 지키고 있는가?
- [ ] 주관식은 **전체 문항의 20% 이하**인가?
- [ ] **총 소요 시간이 8분 이내**인가? (문항당 25초 + 주관식은 45초로 계산)
- [ ] 분기 로직이 필요한 질문은 **분기 조건**이 명확한가?
- [ ] 컨셉/브랜드 설명에 **긍정 유도 표현**이 없는가?
- [ ] 선택지에 **"기타(직접입력)"** 옵션이 포함돼 있는가? (예상 못 한 답변 포착용)
- [ ] **모바일에서 매트릭스형 질문**이 없는가? (있으면 개별 질문으로 분리)
- [ ] NPS 직후 **"왜 그 점수?" 후속 주관식**이 있는가?

### 설계 후 (왜곡 방지 점검)

- [ ] "다 중요하다"고 답할 수 있는 질문이 있는가? → 순위형 or 강제 선택으로 전환
- [ ] 특정 답변을 유도하는 문구가 있는가? → 중립 표현으로 수정
- [ ] 응답자 수가 교차분석 시 **셀당 50명 이상** 확보 가능한가? (최소 30명, 권장 50명)
- [ ] 고정 모듈(M2, M3, M5, M8)의 문항·보기가 이전 서베이와 **완전히 동일**한가?
- [ ] 소요 시간 재계산 — 8분 초과 시 우선순위 낮은 모듈 삭제
- [ ] **파일럿 테스트 10~30명** 완료했는가? (소요 시간, 이해도, 이탈 지점 확인)
- [ ] **주의력 확인 질문(Attention check)** 1개 삽입했는가?

### 분석 시

- [ ] 선택지 기반 결과는 **"선택지에 있었기 때문에 높게 나왔을 가능성"** 감안했는가?
- [ ] 자사 고객 대상 결과를 **시장 전체로 확대 해석**하지 않았는가?
- [ ] PSM 곡선의 **교차점(OPP, 수용 범위)**을 계산했는가?
- [ ] 주관식 답변을 **카테고리화해서 빈도 집계**했는가?
- [ ] 브랜드 퍼널의 **인지→고려 전환율, 고려→사용 전환율**을 계산했는가?
- [ ] NPS를 **추천(9~10) / 중립(7~8) / 비추(0~6)** 3그룹으로 분리했는가?
- [ ] "So what?" — 이 결과를 보고 **다음 행동을 정할 수 있는가?**

## Step 4. 시계열 비교를 위한 규칙

같은 목적의 서베이를 반복할 때, 이전 결과와 비교하려면 아래 규칙을 지켜야 한다.

### 고정해야 하는 것 (절대 변경 금지)

| 항목 | 이유 |
|------|------|
| M2 브랜드 퍼널 — 보조 인지도 브랜드 리스트 | 리스트가 바뀌면 인지도 수치 비교 불가. 추가는 가능, 삭제 금지. |
| M3 구매 결정 요인 — 보기 항목 | 항목이 바뀌면 순위 변화를 추적할 수 없음 |
| M5 PSM — 4문항 문구 및 순서 | Van Westendorp 표준. 바꾸면 곡선 비교 불가. |
| M8 NPS — 문구 및 척도 (0~10) | 산업 표준. 바꾸면 벤치마크 비교 불가. |
| 스크리닝 조건 | 대상이 달라지면 전체 결과 비교 불가 |

### 바꿔도 되는 것

| 항목 | 조건 |
|------|------|
| M4 미충족 니즈 주관식 질문 | 매회 새로운 불만을 포착하기 위해 변경 가능 |
| M7 컨셉 반응 | 매번 다른 컨셉을 제시하므로 당연히 변경 |
| M9 사용 실태 세부 항목 | 카테고리·시점에 따라 조정 가능 |
| M6 채널 보기 | 새 채널 등장 시 추가 가능 (기존 항목 유지하면서) |

## Step 5. 서베이 실행 가이드

### 응답자 수 가이드

| 분석 수준 | 최소 응답자 수 | 근거 |
|----------|-------------|------|
| 전체 결과만 | 200명 | 기본 신뢰도 확보 |
| 성별 or 연령 교차분석 | 300명 | 셀당 50명+ (Kantar/Ipsos 실무 기준) |
| 성별 × 연령 동시 교차분석 | 400~500명 | 셀당 50명+ (2×4=8셀 기준) |
| 자사 인지/비인지 그룹 비교 추가 | 500명+ | 소수 그룹(인지 그룹) 표본 확보 |

- **셀당 50명 미만**: 보고서에 "*참고용" 표시 필수. 30명 미만은 보고 금지.
- **표본 수 계산**: 교차분석 셀 수 × 50 = 필요 응답자 수

### 소요 시간 계산

```
객관식 1문항 = 약 25초
주관식 1문항 = 약 45초
PSM 4문항 = 약 2분
컨셉 제시 + 반응 = 약 2분

권장 총 소요 시간: 8분 이내
7분 초과 시 이탈률 증가 시작
```

### 파일럿 테스트 (본 서베이 전 필수)

- **대상**: 10~30명 (타겟과 유사한 사람)
- **확인 항목**:
  - 실제 소요 시간 (계산값과 차이 확인)
  - 질문 이해도 (오해하는 질문이 있는지)
  - 이탈 지점 (어느 질문에서 그만두는지)
  - 주관식 응답 품질 (의미 있는 답변이 나오는지)
  - 선택지 누락 ("기타"에 같은 답변이 반복되면 보기에 추가)
- **파일럿 후**: 문제 발견 시 수정 → 재파일럿 → 본 서베이

### 모바일 대응 (필수)

응답의 60~70%가 모바일에서 이루어짐. 아래 규칙을 지켜야 함.

- **매트릭스형 질문 금지** → 개별 질문으로 분리. 모바일에서 가독성 최악.
- **한 화면에 1~2문항**. 스크롤 최소화.
- **척도는 5점 권장**. 7점 척도는 모바일 화면에서 너무 좁아 오선택 가능.
- **보기 개수 7개 이하** 권장 (스크롤 없이 한 화면에 보여야 함).
- **주관식 입력창** 크기 충분히 확보.

### 패널 서베이 업체 활용 시 확인사항

- [ ] 타겟팅 조건 (성별/연령/직업군/구매 경험) 설정 가능한지
- [ ] 선택지 랜덤화 기능 지원하는지
- [ ] 분기 로직 (조건부 질문 표시) 지원하는지
- [ ] 순위형 질문 지원하는지
- [ ] 응답 품질 관리 (저품질 응답 필터링) 하는지
- [ ] 주관식 최소 글자 수 제한 설정 가능한지
- [ ] 결과 원본 데이터(Raw Data) 다운로드 가능한지

## Step 6. 분석 가이드

### PSM 곡선 해석법

1. 4개 질문의 응답을 누적 분포로 변환
2. 그래프에 4개 곡선 플로팅:
   - "싸다" 곡선 (낮은 가격에서 100% → 높은 가격에서 0%)
   - "적정" 곡선
   - "비싸다" 곡선 (낮은 가격에서 0% → 높은 가격에서 100%)
   - "절대 안 산다" 곡선
3. 교차점 해석:
   - **OPP (최적 가격)**: "싸다"와 "비싸다" 교차점 → 가격 저항이 최소인 지점
   - **IPP (무관심 가격)**: "적정"과 "절대 안 산다" 교차점
   - **수용 범위**: "너무 싸다"와 "절대 안 산다" 사이

### 브랜드 퍼널 매트릭스

| 지표 | 계산법 | 의미 |
|------|--------|------|
| TOM (최초 상기) | 비보조 1순위 언급 비율 | 카테고리 대표 브랜드 지위 |
| 인지도 | 보조 인지도 응답 비율 | 시장에서 우리를 아는 비율 |
| 인지→숙지 전환율 | 숙지 응답 / 인지 응답 | 이름만 알고 내용은 모르는 비율 → 콘텐츠/노출 부족 |
| 숙지→고려 전환율 | 고려 응답 / 숙지 응답 | 알면서도 안 고려 → 브랜드 이미지/포지셔닝 문제 |
| 고려→구매 전환율 | 구매 경험 / 고려 응답 | 고려했는데 안 삼 → 가격/접근성/제품력 문제 |
| 구매→주사용 전환율 | 주 사용 / 구매 경험 | 사봤는데 반복 안 함 → 만족도/재구매 문제 |

→ **전환율이 낮은 구간 = 마케팅/제품 개선이 필요한 구간**
→ 간소화 시(3단계만 측정): 인지→고려→구매 전환율만 계산

### 주관식 분석 방법

1. 모든 응답을 읽고 **키워드 추출**
2. 유사 키워드를 **5~10개 카테고리**로 그룹핑
3. 카테고리별 **빈도 집계** (응답 수, 비율)
4. 상위 3~5개 = 핵심 인사이트
5. 2차 서베이 시 이 카테고리를 객관식 보기로 전환 → 정량 검증

### 데이터 클리닝 (분석 전 필수)

수집된 응답 중 저품질 데이터를 제거해야 결과 신뢰도가 확보됨.

| 제거 대상 | 기준 | 방법 |
|----------|------|------|
| **빠른 응답 (Speeder)** | 중위 응답 시간의 1/3 이하 | 응답 시간 데이터로 필터링 |
| **일직선 응답 (Straightliner)** | 척도형 질문에서 전부 같은 점수 선택 | 분산이 0인 응답 제거 |
| **주의력 확인 오답** | Attention check 질문에 오답 | 해당 응답 전체 제거 |
| **주관식 무의미 응답** | "ㅇㅇ", "없음", "모름", 1글자 등 | 수동 확인 or 글자 수 필터 |
| **중복 응답** | 동일인 중복 참여 | IP/쿠키/패널ID 기반 제거 |

- **제거 비율 참고**: 전체 응답의 5~15%가 제거되는 것이 일반적
- **제거 후 표본 수 확인**: 클리닝 후에도 교차분석 셀당 50명 이상 유지되는지 확인

## 왜곡 방지 원칙 (상시 참조)

| 왜곡 유형 | 원인 | 방지법 |
|----------|------|--------|
| 선택지 편향 | 보기에 있는 것만 높게 나옴 | 핵심 변수 누락 없이 MECE로 보기 설계 + "기타" 포함 |
| 순서 효과 | 먼저 나온 선택지가 더 많이 선택됨 | 선택지 랜덤화 |
| 유도 질문 | 특정 답변을 암시하는 문구 | 중립적 표현, 긍정/부정 균형 |
| 중앙 경향 | 5점 척도에서 3점에 몰림 | 짝수 척도(4점/6점) 검토 or 강제 선택 |
| 표본 편향 | 자사 고객만 조사 → 시장 전체로 확대 해석 | 대상을 명확히 하고, 해석 시 범위 제한 |
| 복수선택 한계 | "최대 N개" → 변별력 없음 | 순위형으로 전환 |
| 비보조 오염 | 보조 인지도 보기를 먼저 보여줌 | 비보조를 반드시 먼저 배치 |
| 척도 혼란 | 질문마다 척도가 다름 (5점/7점/10점 혼재) | 서베이 내 척도 통일. 모바일은 5점, PC는 5~7점. 양극 척도(매우 불만족~매우 만족)가 표준 |
| 랜덤화 과잉 | 순서가 의미 있는 항목까지 랜덤화 | **랜덤화 예외**: 척도(매우불만~매우만족), 시간순(어제/지난주/지난달), 숫자범위(1만/3만/5만), "기타"/"해당없음"은 항상 고정 위치 |
| 대충 응답 | 보상만 받으려는 무성의 응답 | 주의력 확인 질문 삽입 + 데이터 클리닝(Step 6) |

## 서베이 흐름 순서 (연결 구조)

서베이는 단발이 아니라 **의사결정 흐름에 따라 연결**된다.

```
시장 이해 서베이
→ "고객이 OO을 원한다, 가격은 XX 범위, 경쟁사 빈틈은 △△"
→ 제품 기획

컨셉 검증 서베이
→ "이 컨셉의 구매 의향 X점, 적정 가격 XX만원, 개선 요청 사항 △△"
→ 제품 확정 및 출시

고객 만족 서베이
→ "만족도 X점, 불만 포인트 △△, NPS XX"
→ 제품 개선

브랜드 건강 서베이
→ "인지도 X% → Y%, 고려도 변화, 포지션 변화"
→ 마케팅 전략 조정

(다음 카테고리 진입 시 → 시장 이해 서베이 반복)
```

→ 앞 서베이의 결과가 뒤 서베이의 입력이 됨.
→ 고정 모듈(M2, M3, M5, M8)을 유지하면 각 단계 간 데이터가 연결됨.
