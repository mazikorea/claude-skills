# 제품/서비스 추천 스킬

## 추천 프로세스

### Step 1: 요구사항 파악
사용자의 기존 대화 맥락에서 용도, 예산, 선호를 추론한다. 부족한 정보만 질문.
- 용도 (주 사용 목적)
- 예산 범위
- 필수 조건 vs 있으면 좋은 조건
- 기존에 사용 중인 제품 (있으면)

### Step 2: 후보 선정 + 웹 검색
- 웹 검색으로 2025~2026년 최신 제품 정보 확인
- 후보 3~5개 선정 (너무 많으면 혼란)
- 각 후보의 현재 가격 확인 필수

### Step 3: 비교표 (필수)
모든 추천은 비교표로 시작. 항목:

| 항목 | 포함 내용 |
|------|----------|
| 제품명/모델 | 정확한 모델명 |
| 가격 | 현재 시점 실판가 (웹 검색 확인) |
| 핵심 스펙 | 용도에 직접 관련된 스펙만 |
| 장점 | 이 제품만의 차별점 |
| 단점 | 솔직한 약점 |
| 추천 대상 | 어떤 사람에게 맞는지 |

### Step 4: 최종 1개 추천 (필수)
비교표 후 반드시 "최종 추천"으로 1개를 콕 집어준다. 사용자가 "골라줘"라고 안 해도.

형식:
```
최종 추천: [제품명]
이유: (1~2문장, 사용자 맥락 기반)
가격: XXX원
```

## 규칙
- "둘 다 좋습니다" 같은 모호한 결론 금지. 반드시 1개 선택
- 가격은 반드시 웹 검색으로 현재가 확인. 기억에 의존하지 않음
- 사용자의 개인 정보(체형, 가구 구성, 현재 보유 장비/구독 등)는 Memory를 참조한다. 스킬에 하드코딩하지 않음
- 사용자가 URL을 공유하면 해당 제품을 웹에서 직접 확인 후 답변
